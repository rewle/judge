# judge

CI/CD-гейты для реестра скиллов (Claude Code skills) и конфигов. Каждый PR со
скиллом проходит цепочку гейтов **последовательно**: фейл гейта останавливает
проверку — следующие гейты не запускаются.

## Гейты (в порядке прогона)

| # | Гейт | Детерминизм | Что проверяет | Статус |
|---|---|---|---|---|
| 01 | [static](docs/01_static.md) | детерминированный | frontmatter, длина, пути | реализован |
| 02 | [permissions](docs/02_permissions.md) | детерминированный | tools/MCP против allowlist | реализован |
| 03 | [duplication](docs/03_duplication.md) | v0 детерминированный, v1 (semantic) нет | текстуальные + функциональные дубли | реализован; v1 опционален, требует judge |
| 04 | [rubric](docs/04_rubric.md) | недетерминированный (LLM-judge) | 6 критериев качества + groundedness | реализован, требует judge |
| 05 | [token_efficiency](docs/05_token_efficiency.md) | недетерминированный (behavioral) | токены/латентность vs прирост coverage | реализован, требует judge (только api) |
| 06 | [redteam](docs/06_redteam.md) | статика детерминированная, adversarial-часть нет | устойчивость к prompt injection / расширению прав | реализован, статика без judge, adversarial требует judge |

Гейты 01-02 дешёвые и детерминированные — гонять на каждый push. Гейт 03
v0-часть тоже дешёвая — гонять всегда; v1-часть и гейты 04-06 дороже (нужен
judge) — см. `docs/` по каждому гейту про рекомендованную частоту.

## Структура

```
gates/       — код гейтов (по одному модулю на гейт)
docs/        — контракт каждого гейта: вход, логика, порог, выход
policies/    — allowlist прав, критерии рубрики
examples/    — скиллы-фикстуры, каждый рассчитан на провал конкретного гейта
config.yaml  — порядок гейтов, пороги, пути
run_gates.py — последовательный раннер (fail-fast)
```

## Установка

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

## Настройка судьи (гейты 03(v1)/04-06)

Два бэкенда, выбираются через `JUDGE_BACKEND` (по умолчанию `api`).

### api (по умолчанию, нужен для CI)

Настоящий Anthropic API, платный ключ — креды в репозитории не хранятся,
только env:

```bash
export JUDGE_API_KEY=sk-ant-...        # или ANTHROPIC_API_KEY
export JUDGE_BASE_URL=...              # опционально: свой прокси вместо api.anthropic.com
export JUDGE_MODEL=claude-sonnet-5     # опционально, дефолт см. config.yaml -> judge.model
```

Без ключа гейты 04-05 возвращают `not_configured` (не блокируют цепочку, но и
не пропускают молча — статус явно виден в отчёте). Гейт 05 дополнительно
требует `eval.yaml` рядом со SKILL.md (задача + ожидаемое поведение) — без
него возвращает `skipped`, не `not_configured`. Гейт 06 без ключа всё равно
гоняет статический пред-фильтр (не требует ключа); `not_configured` у него
означает только то, что статика чиста, а adversarial-часть недоступна. Гейт
03 без ключа тоже работает как раньше (только v0, лексика) — v1
(семантическая проверка) молча пропускается, не блокирует и не требует
отдельной настройки сверх той же переменной.

### cli (только для локальной обкатки, без платного ключа)

```bash
export JUDGE_BACKEND=cli
export JUDGE_MODEL=sonnet              # алиас или полное имя модели, см. `claude --help`
# опционально: export JUDGE_CLI_BINARY=claude  (если бинарник не в PATH под этим именем)
```

Под капотом дёргает установленный Claude Code CLI (`claude -p ...
--json-schema ... --no-session-persistence --tools ""`), то есть судья — это
модель текущей подписки/сессии, а не отдельный оплачиваемый API-ключ. Годится
для локальной проверки логики гейтов 03(v1), 04 и 06 (протестировано на
фикстурах: семантический дубль на `spreadsheet-duplicate-finder` /
`dataset-redundancy-detector`, рубрика на `golden-skill`, adversarial-прогон
на скилле с `WebFetch`).

Ограничения:
- **Гейт 05 недоступен** — `claude -p` не даёт offline-эндпоинт подсчёта
  токенов без генерации, а это ядро логики гейта (дешёвый overhead без
  траты токенов). Возвращает `not_configured` с явным объяснением.
- Каждый вызов — отдельный процесс `claude -p` (секунды-десятки секунд), а
  не лёгкий HTTP-запрос — параллельные прогоны (`n_runs=8` в гейте 04)
  ощутимо медленнее и тяжелее, чем через api-бэкенд.
- Не предназначен для CI: использует локальную авторизацию сессии, не
  подходит для воспроизводимого прогона на чужой машине/раннере.

## Запуск

```bash
python3 run_gates.py                       # прогнать все скиллы из examples/skills
python3 run_gates.py --skill golden-skill  # один скилл
```
