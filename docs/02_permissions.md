# Гейт 02 — permissions

**Детерминизм:** полный, без внешних вызовов.
**Реализация:** `gates/g02_permissions.py`
**Политика:** `policies/tools_allowlist.yaml`
**Когда гонять:** каждый push (после 01).

## Модель
Deny-by-default. Три списка в политике:
- `allowed_by_default` — не требуют обоснования (Read, Grep, Glob)
- `requires_justification` — допустимы только если в frontmatter скилла
  есть непустое поле `justification`, объясняющее, зачем инструмент нужен
  именно этой задаче
- `always_flag_for_review` — паттерны (fnmatch), которые ВСЕГДА уходят на
  ручное ревью гейта 06 (`redteam`), даже если formально с justification.
  Например `Bash(*)` (неограниченный shell) или `mcp__*__delete*`.

## Вход
Frontmatter скилла, поле `tools:` (список или CSV-строка) + `justification:`.

## Выход
FAIL, если: инструмент вне всех списков; инструмент из
`requires_justification` без заполненного `justification`; инструмент
подпадает под `always_flag_for_review` (даже с обоснованием — это
сознательный hard-stop, ручное решение не автоматизируется этим гейтом).

## Почему это не судья, а статика
Права декларируются явно в frontmatter — это не то, что требует понимания
смысла текста, поэтому гейт держим детерминированным и дешёвым, а не отдаём
LLM-судье.

## Переиспользование
`evaluate_tools(tools, justification, policy)` — движок политики, вынесен
из `check()` отдельной функцией. Гейт 07 (`docs/roadmap_chains.md`)
прогоняет через неё не только собственные `tools:` скилла, а объединение
по транзитивному замыканию `uses:` — так ловится "A с виду безобидный, но
через `uses: [B]` эффективно получает `Bash(*)` от B", что этот гейт
(видит только один скилл) увидеть не может в принципе.

## Открытый вопрос
Список `tools:` в frontmatter — это то, что скилл *просит* явно, а не
гарантия, что тело скилла не подговаривает модель попросить/использовать
что-то ещё в рантайме — ни через собственные инструменты (зона гейта 06,
redteam), ни через вызов скилла, не задекларированного в `uses:` (зона
гейта 07, "обратная проверка" — спроектирована, не реализована, см.
`docs/roadmap_chains.md`).
