# Гейт 01 — static

**Детерминизм:** полный, без внешних вызовов.
**Реализация:** `gates/g01_static.py`
**Когда гонять:** каждый push.

## Вход
Путь к папке скилла (`SKILL.md` внутри).

## Проверки
- `name`: kebab-case (`^[a-z0-9-]{1,64}$`), не содержит `anthropic`/`claude`,
  не является нетронутым именем-заглушкой из шаблона (`my-skill`,
  `example-skill`, `test-skill`, ...)
- `description`: непустой, ≤1024 символов, содержит триггер использования
  (`когда`/`use when`/`whenever`/...)
- тело: ≤500 строк
- пути в теле: только `/`, без `\`
- нет незаполненных плейсхолдеров шаблона (`{{...}}`, `<ИМЯ_ПОЛЯ>`,
  `TODO`/`FIXME`/`TBD`, буквально скопированные подсказки вида "заполни
  это") — ни в `description`, ни в теле вне code span'ов

## Выход
`GateResult(status, message, details.errors: list[str])`. FAIL, если хотя бы
одна проверка не прошла — все нарушения перечисляются в одном сообщении.

## Защита от новичков (реализовано)
Этот гейт — первое, что видит человек, впервые пишущий SKILL.md, часто без
понимания формата вообще. Поэтому здесь два целенаправленных решения:

1. **Сообщения об ошибках с примером** ("name не kebab-case" →
   + "пример правильного: 'my-skill-name'"), а не просто констатация факта.
2. **Проверка плейсхолдеров.** Если человек скопировал шаблон и не заменил
   `{{...}}`-подсказки на реальный текст — раньше это тихо проходило гейт 01
   (валидный, но пустой по содержанию скилл) и обнаруживалось бы только на
   дорогом гейте 04 (рубрика), если вообще обнаруживалось. Теперь ловится
   сразу, бесплатно, с явным списком, что именно осталось заполнить.
3. **`new_skill.py`** (в корне репозитория) — генератор скелета: валидный
   frontmatter и структура по конструкции, реальный текст — через
   `{{...}}`-плейсхолдеры, которые гейт 01 не пропустит, пока не заменены.
   `run_gates.py --path <папка>` позволяет проверить скелет сразу вне
   `skills_dir` (не обязательно класть его в реестр, чтобы просто
   протестировать).
4. **`_read_skill` не роняет весь раннер на невалидном YAML.** Раньше
   незаэкранированные спецсимволы в `description` (например `{{...}}` без
   кавычек — ровно то, что писал бы новичок вручную) приводили к
   необработанному `yaml.YAMLError` и трейсбеку вместо понятного `FAIL`.
   Обнаружено вживую при тестировании `new_skill.py`, не гипотетически.

## Известные ограничения v0
- Проверка триггера — по подстроке, не по смыслу. Description на триггерное
  слово, но не по делу, пройдёт проверку. Смысловую оценку description
  (говорит ли он одновременно "что" и "когда") делает рубрика (гейт 04).
- Плейсхолдер-паттерны эвристические (список в коде, не политика в yaml,
  в отличие от гейта 06) — новый формат шаблона с другим синтаксисом
  подсказок этот список не поймает, придётся дополнять `PLACEHOLDER_PATTERNS`
  вручную.
