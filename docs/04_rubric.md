## Критерии (0.0–1.0 каждый)
Существующие 6 из ноутбука:
- conciseness, description_quality, degrees_of_freedom, concrete_examples,
  consistent_terminology, actionable_workflow

Новый для этого проекта:
- **groundedness** — содержит ли тело утверждения, которые нельзя
  проверить (придуманные факты, флаги, API), поданные уверенно, без пометки
  "не проверено"/"уточнить"; инструктирует ли скилл модель угадывать вместо
  проверки реального состояния (см. фикстуру `hallucination-prone-skill`)

## Прогон
8 запусков судьи на каждый критерий (как в существующей методологии) →
среднее + sigma. Fail если `mean < thresholds.rubric_min_score` ИЛИ
`sigma > thresholds.rubric_max_sigma` (нестабильность судьи — тоже сигнал,
не игнорировать).

## Открытые вопросы для реализации
- Какая модель — судья (дешевле/быстрее, чем модель разработки скилла, но
  не настолько слабая, чтобы сама галлюцинировала оценку)
- Хранить ли историю прогонов рубрики по каждому скиллу (дрифт судьи со
  временем при обновлении модели)
- Порог `rubric_min_score=0.7` — взят по аналогии с эталоном
  `ingest-dlq-analysis` (rubric 0.87), нужно откалибровать на большей выборке
