# CLAUDE.md

Полная инструкция по проекту — в [AGENTS.md](AGENTS.md), она не дублируется
здесь. Прочитать перед любой работой в этом репозитории.

## Модель

Использовать Sonnet 5 (`claude-sonnet-5`), как и в остальных проектах
владельца.

## Специфично для Claude Code

`judge_client.py` умеет бэкенд `JUDGE_BACKEND=cli`, который сам шеллится в
`claude -p ...` — то есть у этого репозитория есть собственный «встроенный»
вызов Claude Code. Если сессия Claude Code работает над этим репозиторием и
одновременно тесты/гейты гоняют cli-бэкенд, это два независимых процесса
(текущая сессия и порождённые `claude -p`), они не делят контекст и не
мешают друг другу технически, но могут удивить по времени выполнения —
каждый вызов cli-бэкенда — это отдельный процесс claude (секунды-десятки
секунд), не лёгкий HTTP-запрос. Не путать его с самим собой.

## Правило: не додумывать неясности

Если хоть что-то непонятно — любая мелочь, — не додумывать и не предполагать
по умолчанию. Лучше переспросить пользователя 10 раз лишний раз, чем потом
10 раз всё переделывать. Действует для этого репозитория в целом (не только
для конкретной задачи).
