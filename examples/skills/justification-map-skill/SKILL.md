# Отчёт по воронке из DWH

Положительный контроль гейта 02 для двух решённых осей сразу
(`docs/design_multisystem_groups.md`):

- инструменты `mcp__dwh__*` разрешены не глобальной политикой, а файлом
  системы `policies/systems/dwh.yaml` (решение 1C, включая паттерн
  `mcp__dwh__export_*` в requires_justification);
- `justification` здесь карта {инструмент: причина}, а не строка
  (решение 3B): обоснование доступа к хранилищу и обоснование записи
  файла — разные причины, и ревьюер видит каждую.

## Шаг 1. Собрать агрегаты

Выполнить агрегирующий запрос по стадиям воронки через `mcp__dwh__query`
за запрошенный период.

## Шаг 2. Выгрузить сегмент

Если аналитик просил детализацию, выгрузить таблицу сегмента через
`mcp__dwh__export_table`.

## Шаг 3. Сохранить отчёт

Сформировать сводку по конверсии стадий и записать одним файлом
`funnel-report.md` через Write.
