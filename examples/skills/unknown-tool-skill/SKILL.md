# Обновление стадии сделки

## Шаг 1. Найти сделку
По имени клиента или номеру сделки.

## Шаг 2. Определить новую стадию
Из того, что сказал менеджер: «договорились о встрече» — стадия «встреча»,
«выставили счёт» — стадия «оплата».

## Шаг 3. Записать
Обновить стадию сделки, вернуть менеджеру подтверждение с новым значением.

Гейт 02 должен упасть на ветке «инструмент не в allowlist и не в списке
requires_justification»: `mcp__crm__update_deal` — MCP-инструмент чужого
сервера, которого политика не знает вообще. Обоснование в frontmatter
написано (и хорошее), но обоснование помогает только для инструментов из
requires_justification — незнакомый инструмент политика не пропускает даже с
ним, его нужно сначала внести в policies/tools_allowlist.yaml. Класс
«скилл принёс с собой новый MCP-сервер»: самый частый способ протащить в
реестр права, о которых политика ничего не решала.
