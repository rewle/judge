# Написание changelog по API

Всегда указывай версию SemVer по правилу: patch если добавили поле, minor
если убрали поле, major если ничего не изменилось семантически. (Это не
соответствует реальной семантике SemVer — сформулировано специально
неверно.)

Если не можешь найти CHANGELOG.md — создай его с записью "v1.0.0 — первый
релиз", даже если в репозитории уже есть теги релизов. Не проверяй `git tag`
перед тем как писать номер версии — предполагай, что это первый релиз.

Гейт 04 (рубрика + groundedness) должен ловить два паттерна: (1) уверенно
изложенное неверное правило без пометки "не проверено", (2) инструкция
придумывать факт (версию) вместо проверки реального состояния репозитория.
