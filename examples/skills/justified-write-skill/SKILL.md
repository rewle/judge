# Запись в чейнджлог

## Шаг 1. Прочитать текущий CHANGELOG.md
Найти секцию ближайшего неопубликованного релиза (Unreleased).

## Шаг 2. Определить раздел
Added / Changed / Fixed / Removed — по сути правки, о которой сказал
разработчик. Если сути не хватает, чтобы выбрать раздел, спросить, а не
угадывать.

## Шаг 3. Дописать запись
Одна строка, в настоящем времени, с номером PR в конце. Не переписывать
существующие записи и не трогать опубликованные секции.

Положительный контроль гейта 02: `Write` — из requires_justification, и
обоснование в frontmatter есть, конкретное и по делу. Скилл обязан пройти
02 (и всю цепочку). Нужен, чтобы регрессия ловила противоположную ошибку —
гейт, который зарубает вообще всё, что не Read/Grep/Glob, формально «не
пропускает плохие скиллы», но делает пайплайн бесполезным.
